Our best configuration utilizes a 1.1 million parameter GPT architecture paired with a custom Byte-Pair Encoding (BPE) tokenizer featuring a 1,024 vocabulary size. The model architecture consists of 5 transformer layers, 4 attention heads, an embedding dimension of 128, and a context block size of 128. This specific configuration thrives because the BPE tokenizer compresses the mixed English/Hindi UTF-8 corpus by roughly 46%. This effectively doubles the semantic information packed into the 128-token context window without incurring the quadratic computational cost of larger self-attention blocks. Keeping the parameter count near 1.1M explicitly prevents CPU thread thrashing and memory bottlenecks on local Windows hardware. By tying the embedding and language modeling head weights, we successfully reallocated our strict 2,000,000 parameter budget directly into the transformer blocks for deeper reasoning. Furthermore, utilizing manual tensor slicing rather than standard `DataLoader` multiprocessing sidestepped native Windows deadlocks, ensuring the model could complete all 2,000 steps efficiently. This careful balance of custom data compression and architectural restraint yields the optimal Bits-Per-Byte (bpb) performance.
